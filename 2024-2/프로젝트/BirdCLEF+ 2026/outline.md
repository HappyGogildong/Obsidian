# Executive Summary  
BirdCLEF+ 2026 대회는 브라질 파나마(Pantanal) 습지의 **야생동물 음향 기록**에서 조류·양서류·포유류·파충류·곤충 등 다양한 종을 식별하는 머신러닝 모델 개발 경진대회다. 본 보고서는 대회 개요(설명, 일정, 규칙, 제출 형식, 평가 지표, 허용 리소스, 팀 크기, 상금, Kaggle 환경)와 대회 목표(타겟 과제, 출력 형식, 동기, 성공 기준)를 정리하고, 제공 데이터셋의 구조(파일/폴더 목록, 형식, 샘플 수, 라벨·메타데이터 필드, 오디오 포맷/샘플링, 주어진 train/valid/test 분할 및 클래스 불균형)와 EDA/전처리 고려사항을 상세 분석한다. 특히 **1분 길이 녹음(audio)**을 **5초 구간**으로 분할해 수백 종의 존재 확률을 예측하는 과제로, 평가 지표는 **보정된 다중 레이블 macro-ROC-AUC**이다【53†L1-L4】. 분석을 통해 데이터 구조(파일 유형, 크기, 행 수, 목적 비교표 포함)와 예비 EDA/전처리 방안을 제시한다.  

## 1. 대회 및 환경 개요  
- **대회 설명:** BirdCLEF+ 2026는 파나마 습지에 설치된 방음 기록기에서 수집된 음성 데이터를 이용해 야생동물(특히 조류)을 자동 식별하는 대회다【6†L79-L87】. 광범위한 생물다양성 모니터링을 지원하기 위해 음향 센서 네트워크를 활용하며, 결과물은 생태계 보전과 조류 개체수 추적 등에 활용된다
- **일정:** 2026년 3월 11일 대회 시작, 5월 27일 제출 마감(참가등록/팀 합류 마감 포함), 6월 3일 최종 제출 마감이다. (이후 CLEF 학회 워킹노트 제출 기한도 별도로 6월 17일까지 제공된다.)  
- **규칙:** Kaggle 규칙에 따라 진행되며, 외부 데이터 및 사전학습 모델 사용은 명시적 금지사항이 없는 한 허용된다. 팀원 수 최대 5명(Kaggle 표준)이며, 하루 최대 5회, 최종 제출은 2회까지 허용된다. 제출은 Kaggle 제공 템플릿(초기 샘플 submission)에 맞춰 수행한다.  
- **제출 형식:** 테스트 집합의 **각 5초 구간(row_id)**별로 **모든 종(Classes) 출현 확률**을 예측해 제출한다. 즉, 컬럼마다 각 종의 확률값을 기록하는 **다중레이블(probability)** 방식이다. (지정된 234개 종이 타겟이며, 각 행은 5초 단위 소리 구간에 해당한다.)  
- **평가 지표:** 보정된 **macro-averaged ROC-AUC** (진짜 양성 클래스만 집계)에 따라 점수가 매겨진다【53†L1-L4】. 다중 레이블 분류의 ROC-AUC를 종별 평균으로 계산하되, 참 양성(label=1)이 전혀 없는 클래스를 제외한다.  
- **허용 리소스:** Kaggle 노트북 환경에서 Python, R 등 모든 라이브러리(TensorFlow, PyTorch, scikit-learn, librosa 등) 사용 가능하며, 공개된 외부 데이터나 사전학습 모델도 규정에 저촉되지 않으면 활용할 수 있다
- **Kaggle 환경:** Kaggle Notebooks(커널)는 GPU(Tesla T4/P100/최근 A100 등)와 TPU를 제공하며, 무료 사용 시간(주당 GPU 30시간 제한, 각 세션 9시간 실행제한, 유휴 60분 타임아웃) 제약이 있다. 데이터는 제공된 Kaggle 경로(`/kaggle/input/...`)에서 사용 가능하며, 오디오 파일(.ogg) 처리를 위해 librosa, torchaudio, soundfile 등의 패키지가 필요하다.  

## 2. 대회 목표  
- **타겟 과제:** 주어진 **1분 길이의 현장 녹음(soundscape)**에서 **5초 윈도우** 단위로 여러 종의 출현 유무를 예측하는 **다중레이블 분류** 문제다. 녹음 한 개(파일)는 5초 구간 12개로 나뉘며, 각 구간에 대해 650여 종 중 어느 종들의 소리가 포함되었는지 확률로 출력해야 한다. 대상 동물군은 조류뿐 아니라 양서류, 파충류, 포유류, 곤충 등으로 다양하며(총 234개 종)【53†L1-L4】, 한 녹음에 여러 종이 동시에 등장할 수 있다. 주어진 라벨(교재) 파일에는 각 5초 구간별로 출현한 종 목록(공백 구분, 발생하지 않으면 “nocall”)이 기록된다.  
- **예상 출력:** 제출물은 CSV 형식으로, 각 행은 `(row_id, species_1_prob, species_2_prob, ..., species_234_prob)`를 포함한다. 즉, 각 5초 단위 `row_id`에 대해 234개 종 각각의 출현 확률을 제출한다. (과거 BirdCLEF과 달리 다중확률 포맷을 사용한다.)  
- **비즈니스/과학적 동기:** 광활한 자연환경에서 야생동물을 모니터링하기 위해 음향 센서를 활용하는 자동화 기술 개발이 목표다. 파나마 습지는 150,000km² 이상의 광범위한 습지로, 650종 이상의 조류가 서식하는 세계 최대 규모의 생물다양성지이다. 그러나 계절 변화, 산불, 농업개발, 기후변화 등으로 현장 조사가 어렵다. 이 대회는 ‘보이지 않는 생태계를 듣기(Listening)’ 위해 개발된 기계학습 기반 도구를 통해, 연구자들이 넓은 지역의 생물다양성 변화를 효율적으로 추적·분석할 수 있도록 돕는다
- **성공 기준:** 다중 레이블 ROC-AUC 등 정량 지표에서 높은 점수를 얻는 것이 성공 기준이다. 특히 **macro-averaged ROC-AUC**(각 종별 AUC를 평균) 점수가 상위권(메달권)으로 평가된다. 대회 성격상 검증 데이터가 없으므로 제출 순위(LB)와 수상 여부가 곧 성과를 판가름한다.

### 데이터 EDA 및 전처리 고려사항  
- **클래스 분포 분석:** 메타데이터와 라벨 파일을 이용해 종별 녹음 수, 음향 구간 수를 파악한다. (예: 특정 종이 소수의 샘플만 가지며, 다수는 수백 개를 가질 수 있음. `primary_label`의 분포를 그래프로 시각화.) 불균형 확인 후, 소수 클래스에 대한 증강 또는 손실함수(포컬 손실, 비대칭 손실 등) 고려가 필요하다.  
- **지리·시계열 분포:** `latitude`, `longitude`, `date` 필드를 활용해 녹음 위치와 시기별 데이터 분포를 확인한다. 일부 종은 특정 지역/계절에만 출현할 수 있으므로 모델 학습 시 지역·계절적 다양성을 확보해야 한다.  
- **오디오 길이 및 형식:** `train_audio`의 녹음 길이는 짧음(몇 초~수십 초까지 가변). `train_soundscapes`와 `test_soundscapes`는 고정 60초 길이이며, 5초 창으로 분할한다【53†L1-L4】. 모든 오디오는 OGG 형식으로, 훈련 데이터를 검색해보면 샘플링 레이트가 32,000Hz로 통일된 것으로 보인다(불명시, BirdCLEF 전대회 기준으로 32kHz 예상). 전처리 시 OGG→WAV 변환과 리샘플링(필요시) 수행한다.  
- **특징 추출:** 각 5초 구간에 대해 멜-스펙트로그램, MFCC, 페커트럼 등으로 변환하고 정규화한다. Librosa나 torchaudio 라이브러리 사용이 일반적이다.  
- **데이터 증강:** 잡음(백그라운드 소리) 주입, 타임 시프트(시간 축 롤링), 주파수/시간 마스킹(SpecAugment) 등을 활용해 모델 일반화 성능을 높인다. 예를 들어, 상위 솔루션에서는 **Waveform mixup** (최대 3개 소스 혼합), **주파수/시간 마스킹**, **배경 잡음 추가**(SNR 3-30dB), **에너지 기반 구간 선택** 등을 적용했다【11†L70-L78】.  
- **학습 설정:** 훈련 시에는 5초 구간 단위(또는 겹치도록 슬라이딩)로 미니배치 구성, 배치 크기와 학습률, 옵티마이저 등은 하이퍼파라미터로 조정한다. 앙상블 학습 시 5-플릿 교차검증이 흔히 사용된다. 대회 상위권 솔루션은 ViT 기반의 Bird-MAE, EfficientNet-B 등 사전학습 모델을 도메인 적응해 사용했다【11†L57-L65】.  
- **초기 EDA/전처리:** CSV 메타데이터를 로드하여 결측치 유무, 중복 확인 후, 종(class) 수, 각 클래스의 샘플 수, 녹음 길이(구간 수) 분포를 시각화한다. `train_soundscape_labels.csv`는 5초 윈도우별로 라벨이 있으므로 이를 기준으로 타겟 종을 확인하고, 훈련 시 활용 가능한 라벨 세부정보를 파악한다. 각 오디오 파일의 채널 수(스테레오/모노) 확인 후, 필요시 단일 채널 변환을 검토한다.
