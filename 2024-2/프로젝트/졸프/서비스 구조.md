**Offline**
TEXT_CANDIDATE → CLIP → text embedding (normalize) → 저장

**Online**
이미지 → CLIP → image embedding (normalize)
→ 모든 text embedding과 cosine (≒ dot) 비교
→ Top-K landmark 반환

# 2. 반드시 지켜야 할 3가지 (이거 안 하면 성능 무너짐)

## (1) 텍스트를 “한 개”로 두지 마라

각 랜드마크 = 여러 문장

```
경복궁:- "경복궁"- "서울 경복궁 궁궐"- "Gyeongbokgung Palace in Seoul"- "조선 왕궁 경복궁"
```

👉 **prompt ensemble 효과**

---

## (2) 반드시 정규화

```
x = x / ||x||y = y / ||y||score = x @ y
```

👉 안 하면 similarity 왜곡됨

---

## (3) Top-K 후 aggregation

```
Top-5 text 결과:경복궁, 경복궁, 창덕궁, 경복궁, 덕수궁→ 경복궁
```

또는:

```
score weighted sum
```

# 3. 성능 끌어올리는 핵심 트릭 (여기서 차이 난다)

```

👉 여러 템플릿 평균

경복궁 + Gyeongbokgung
```
