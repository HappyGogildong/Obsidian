# static
## 1. 핵심 개념

```
class Example {    static int a = 10;    int b = 20;}
```

- `a` → 클래스 변수 (static 변수)
- `b` → 인스턴스 변수

접근 방식:

```
Example.a      // O (클래스 이름으로 접근)new Example().b // O (객체 필요)
```

👉 `static`은 **객체 생성 없이 접근 가능**하다는 게 핵심

---

## 2. static의 적용 대상

### (1) static 변수 (Class Variable)

```
class Counter {    static int count = 0;    Counter() {        count++;    }}
```

- 모든 객체가 **공유**
- 메모리 상에 **하나만 존재**

---

### (2) static 메서드

```
class MathUtil {    static int add(int a, int b) {        return a + b;    }}
```

사용:

```
MathUtil.add(1, 2);
```

특징:

- 인스턴스 없이 호출 가능
- **instance 변수 접근 불가**

```
static void test() {    // this.b  ❌ (인스턴스 없음)}
```

---

### (3) static 블록

```
class Init {    static {        System.out.println("클래스 로딩 시 실행");    }}
```

- 클래스 로딩 시 **딱 한 번 실행**
- 초기화 로직에 사용

---

### (4) static 클래스 (nested class)

```
class Outer {    static class Inner {        void print() {            System.out.println("static inner");        }    }}
```

---

## 3. static vs 접근 지정자

|구분|역할|
|---|---|
|`public / private / protected`|접근 범위 제어|
|`static`|소속 대상 (클래스 vs 객체)|

👉 완전히 다른 개념

---

## 4. 메모리 관점 (중요)

- static → **Method Area (또는 Metaspace)**
- instance → **Heap**

즉:

```
static → 프로그램 전체에서 공유instance → 객체마다 별도
```

---

## 5. 자주 나오는 제약

### (1) static → instance 접근 불가

```
class Test {    int x = 10;    static void foo() {        // System.out.println(x); ❌    }}
```

이유:

- static은 객체 없이 실행됨 → instance 존재 보장 안 됨

---

### (2) instance → static 접근 가능

```
void bar() {    System.out.println(Test.a); // 가능}
```

---

## 6. 실전 감각

### 언제 static 쓰냐?

- 상태 공유 필요할 때  
    → ex) 카운터, 설정값
- 유틸 함수  
    → ex) `Math`, `Collections`
- 객체 생성 비용 줄일 때

---

## 7. 한 줄 정리

> `static`은 “접근 범위”가 아니라  
> **“객체가 아니라 클래스에 귀속시킨다”**는 개념이다.
