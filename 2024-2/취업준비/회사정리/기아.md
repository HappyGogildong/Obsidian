#### 인턴 면접 질문
### [제한된 자원을 돌파하는 기술적 유연성: 지식 증류 모델의 학습 최적화]

**[문제 상황: 하드웨어 한계와 학습 불안정]** 2026 저전력 컴퓨터 비전 경진대회(LPCVC)에 참여하며 MobileNetV4 기반의 경량 CLIP 모델을 구축하는 과정에서 기술적 한계에 부딪혔습니다. 고성능 SigLip2와 ViT-L-14 모델을 교사(Teacher) 모델로 활용하는 '이중 지식 증류'를 기획했으나, 보유한 GTX 4070(12GB VRAM) 환경에서는 대형 모델 두 개를 동시에 로드할 때 메모리 부족(OOM) 에러가 발생했습니다.

**[해결 방안: 전략적 선택과 공학적 접근]** 저는 이 문제를 단순히 하드웨어 탓으로 돌리지 않고, 두 가지 단계적 최적화 전략을 통해 해결했습니다.

**첫째, 성능과 복잡도의 트레이드오프를 고려한 모델 전략 수정입니다.** 서로 다른 손실 함수와 분포를 가진 두 교사 모델을 동시에 학습시키는 것은 자원 소모뿐만 아니라 학습 난도 자체를 높이는 요인이었습니다. 이에 저는 일반화 성능과 학습 효율을 재검토하여, 상대적으로 우수한 성능을 보인 SigLip2를 단일 교사 모델로 선정함으로써 메모리 점유율을 낮추고 학습의 수렴 속도를 높였습니다.

**둘째, Gradient Accumulation 기법을 통한 학습 안정성 확보입니다.** 메모리 제약으로 인해 배치 사이즈를 4까지 줄여야 했으나, 작은 배치 사이즈는 그래디언트 노이즈를 발생시켜 학습을 불안정하게 만들었습니다. 이를 해결하기 위해 32 스텝의 그래디언트 누적 기법을 적용했습니다. 이를 통해 실질적으로 128 배치 사이즈와 동일한 가중치 업데이트 효과를 얻었으며, 한정된 VRAM 안에서도 안정적인 손실값 하락을 이끌어낼 수 있었습니다.

**[결과 및 교훈: 자원 효율의 극대화]** 이러한 공학적 시도를 통해 개인 PC 환경에서도 대규모 지식 증류 학습을 성공적으로 구현해냈습니다. 초기에는 1 epoch당 30시간에 달하는 시간이 소요되었으나, 로컬 환경에서 검증된 최적화 파이프라인을 바탕으로 학교 GPU 자원을 효율적으로 활용하여 프로젝트를 기한 내 완수할 수 있었습니다.

이 경험을 통해 기술의 화려함보다 중요한 것은 **'주어진 자원 내에서 최선의 결과를 내기 위한 유연한 전략'**임을 배웠습니다. 어떤 환경에서도 문제의 본질을 파악하고 기술적 대안을 찾아내는 개발자가 되겠습니다.



GTX 4070  GPU를 이용하여 SigLip2 모델, ViT-L-14-로터 mobileclip2 모델을 학습시키는 지식 증류 방식의 학습을 진행하던 중  GPU 메모리가 부족해 Out of Memory에러가 발생했었습니다.  파라미터 사이즈가 큰 두 선생 모델을 메모리에 모두 올릴 수 없었던 탓이었습니다. 

첫 번째로, ViT-L-14 모델을 제외했습니다. 
학습 데이터가 다른 두 선생 모델을 사용해 일반화 성능을 올리고자 했지만, 두 선생 모델의 loss 함수가 다르고, 분포가 달라 학습 난이도가 높았습니다. 따라서 두 모델 중 하나를 제외하고자 했고 ViT-L-14 모델을 제외했습니다. 하지만, SigLip2 단일 모델도 파라미터 사이즈가 커 원활한 학습이 진행되지 않았습니다.

두 번째로, gradient accumulation step을 적용했습니다. 
GPU 메모리가 낮아 배치 사이즈를 4로 줄였는데, 배치 사이즈가 너무 작으면 노이즈가 커 학습이 불안정해집니다.
따라서 gradient accumulation step을 32로 적용해 128 배치 사이즈만큼 누적했다가 파라미터를 수정하게 바꾸어 
더 안정적인 학습을 할 수 있게 하였습니다.

이러한 시도를 통해 개인 PC 환경에서도 대규모 모델의 지식 증류 학습을 구현해낼 수 있었습니다. 1epoch에 30시간 정도가 소요되어 학습의 진행 가능 여부를 점치는 정도에 그쳤으나, 학교 GPU를 대여해 학습을 진행해 프로젝트를 기간 내 완수할 수 있었습니다.

**학습 내용** : **지식 증류, gradient accumulation step, CLIP, ViT, SigLip2**

학부에서 기본적인 프로그래밍 지식을 습득한 이후, 실제 서비스 개발 경험을 쌓기 위해 2025년 4월부터 11월까지 카카오테크캠퍼스 활동을 진행했습니다. Spring, AWS와 같이 백엔드 개발에 필요한 기술을 배우고, 팀을 구성해 프로젝트를 진행했습니다. 프로젝트명은 펫위워크로, 반려견 산책로 추천 앱입니다. 

저는 프로젝트에서 사용자 및 반려견 도메인을 담당했습니다.

먼저 Spring Security를 활용하여 JWT 기반 인증,인가 체계를 구축했습니다. 이 과정에서 인증과 인가의 차이에 대해 
이해하고, CORS 설정과 필터 체인을 포함한 요청 처리 흐름을 완성하여 안정적인 로그인 및 API 접근을 구현했습니다.

또한 반려견 및 사용자 정보를 관리하는 CRUD API를 RESTful하게 구현하기 위해 HTTP 메서드와 상태 코드를 명확히 구분하였습니다. 
깃허브를 활용한 협업 과정에서 Git flow 브랜치 전략과 코드 리뷰를 통해 코드 품질을 개선하며 팀 개발 프로세스를 경험했습니다. 

**학습 내용 : Spring Filter, Interceptor, Spring, JWT, CORS, CSRF, RESTFul, HTTP, Git Flow**
