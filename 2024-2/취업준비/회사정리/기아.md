#### 인턴 면접 질문
### [제한된 자원을 돌파하는 기술적 유연성: 지식 증류 모델의 학습 최적화]

**[문제 상황: 하드웨어 한계와 학습 불안정]** 2026 저전력 컴퓨터 비전 경진대회(LPCVC)에 참여하며 MobileNetV4 기반의 경량 CLIP 모델을 구축하는 과정에서 기술적 한계에 부딪혔습니다. 고성능 SigLip2와 ViT-L-14 모델을 교사(Teacher) 모델로 활용하는 '이중 지식 증류'를 기획했으나, 보유한 GTX 4070(12GB VRAM) 환경에서는 대형 모델 두 개를 동시에 로드할 때 메모리 부족(OOM) 에러가 발생했습니다.

**[해결 방안: 전략적 선택과 공학적 접근]** 저는 이 문제를 단순히 하드웨어 탓으로 돌리지 않고, 두 가지 단계적 최적화 전략을 통해 해결했습니다.

**첫째, 성능과 복잡도의 트레이드오프를 고려한 모델 전략 수정입니다.** 서로 다른 손실 함수와 분포를 가진 두 교사 모델을 동시에 학습시키는 것은 자원 소모뿐만 아니라 학습 난도 자체를 높이는 요인이었습니다. 이에 저는 일반화 성능과 학습 효율을 재검토하여, 상대적으로 우수한 성능을 보인 SigLip2를 단일 교사 모델로 선정함으로써 메모리 점유율을 낮추고 학습의 수렴 속도를 높였습니다.

**둘째, Gradient Accumulation 기법을 통한 학습 안정성 확보입니다.** 메모리 제약으로 인해 배치 사이즈를 4까지 줄여야 했으나, 작은 배치 사이즈는 그래디언트 노이즈를 발생시켜 학습을 불안정하게 만들었습니다. 이를 해결하기 위해 32 스텝의 그래디언트 누적 기법을 적용했습니다. 이를 통해 실질적으로 128 배치 사이즈와 동일한 가중치 업데이트 효과를 얻었으며, 한정된 VRAM 안에서도 안정적인 손실값 하락을 이끌어낼 수 있었습니다.

**[결과 및 교훈: 자원 효율의 극대화]** 이러한 공학적 시도를 통해 개인 PC 환경에서도 대규모 지식 증류 학습을 성공적으로 구현해냈습니다. 초기에는 1 epoch당 30시간에 달하는 시간이 소요되었으나, 로컬 환경에서 검증된 최적화 파이프라인을 바탕으로 학교 GPU 자원을 효율적으로 활용하여 프로젝트를 기한 내 완수할 수 있었습니다.

이 경험을 통해 기술의 화려함보다 중요한 것은 **'주어진 자원 내에서 최선의 결과를 내기 위한 유연한 전략'**임을 배웠습니다. 어떤 환경에서도 문제의 본질을 파악하고 기술적 대안을 찾아내는 개발자가 되겠습니다.
